# Presentation Design Spec Rule

Use this rule when NDT receives a KAT handoff or production packet and must define the visual system before producing a PPT, web deck, image deck, or video-adjacent presentation.

## `design_spec`

The design spec records:

- audience and meeting context;
- selected route and subroute;
- visual posture;
- page system and slide families;
- typography hierarchy;
- grid, margin, spacing, and density;
- chart treatment;
- proof-object treatment;
- image direction and gpt-image-2 boundary;
- accessibility and readability constraints;
- output engine and QA requirements.

The design spec must not rewrite KAT conclusions, numbers, source notes, or regulatory wording.

## `style_lock`

The style lock records:

- approved colors and token references;
- typography scale;
- title/action-title behavior;
- chart palette and label rules;
- image treatment and prohibited motifs;
- component/panel patterns;
- allowed variation range;
- elements that require return-to-KAT or NERO approval.

Use style lock after the visual direction is selected. Do not lock style before checking KAT content constraints.

## Three-Direction Exploration

For important decks, create three visual directions before final production:

1. Conservative banker: restrained, dense, evidence-first.
2. Editorial strategy: stronger hierarchy and narrative pages.
3. Modern technical: architecture/data/product-forward, still professional.

Each direction must include:

- use case;
- avoid case;
- key slide treatment;
- token references;
- image direction;
- risk notes;
- selection rationale.

This can be text-only for early planning or a contact sheet for visual work. Do not overproduce visual variants when the task is audit-only or time-critical.

## gpt-image-2 Boundary

Use gpt-image-2 only for covers, chapter visuals, industry concept scenes, backgrounds, or exploration素材.

Never use generated images for exact body text, Chinese conclusions, financial numbers, tables, chart axis labels, source notes, regulatory wording, or official claims.

## PPTX Readiness

Before reporting a formal PPTX as ready, verify or explicitly mark unverified:

- editable text/shapes where required;
- slide screenshots/previews;
- text overflow and overlap;
- chart readability;
- source notes and exact data preservation;
- content freeze gate compliance;
- production-check status.
